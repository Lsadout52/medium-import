Introduction to Google Cloud Storage

Google Cloud Storage is a scalable service for storing and retrieving files in any format. It is commonly used as a staging area for data that needs to be accessed by services like Cloud SQL, BigQuery, and Dataproc. It also serves as a content delivery solution, helping businesses store and serve media, backups, and large datasets efficiently.

Getting Started

Creating a Bucket

A bucket is a storage container where your files reside. To create one:

  • Open Google Cloud Console and navigate to Cloud Storage.
  • Click Create Bucket, give it a unique name, and select a storage class.
  • Choose a location and set permissions.
Alternatively, using gsutil: 

gsutil mb gs://my-new-bucket/

Buckets can also be configured with object versioning to retain previous versions of files, which is useful for rollback and data recovery.

Uploading and Managing Files

To upload a file: 

gsutil cp file.ext gs://my-new-bucket/

To upload multiple files efficiently:

gsutil -m cp file*.ext gs://my-new-bucket/

To delete a file:

gsutil rm gs://my-new-bucket/file.ext

To list files in a bucket:

gsutil ls gs://my-new-bucket/

Storage Classes

Google Cloud Storage offers different classes based on access needs:

  • Standard - Frequently accessed data.
  • Nearline - Accessed less than once a month.
  • Coldline - Long-term storage, rare access.
  • Archive - Regulatory backups, lowest cost.

To create a Nearline bucket:

gsutil mb -c nearline gs://my-nearline-bucket/

Using Cloud Storage for Websites

Google Cloud Storage can serve static websites, eliminating the need for a traditional web server. Steps:

  • Create a bucket named after your domain (e.g., example.com).
  • Upload website files.
  • Make them public.
  • Configure it for static hosting:

gsutil web set -m index.html gs://example.com/

Use a CDN or domain name system (DNS) configuration to serve content efficiently worldwide.

Access Control and Security

Cloud Storage provides fine-grained access controls to keep data secure:

  • IAM Roles: Assign roles to control who can view or modify objects.
  • Object ACLs: Customize access at the file level.
  • Encryption: Data is automatically encrypted at rest and in transit.

To grant public read access to a file:

gsutil acl ch -u AllUsers:R gs://my-new-bucket/public-file.ext
